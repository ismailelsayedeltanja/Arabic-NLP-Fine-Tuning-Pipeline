# Fine-tuning
Fine-tuning and evaluation pipeline for Arabic NLP tasks using Qwen 2.5, Groq, OpenAI, and LLaMA-Factory. The project focuses on structured information extraction, machine translation, synthetic dataset generation, supervised fine-tuning (SFT), LoRA training, model evaluation, and scalable deployment with vLLM

---

# Arabic NLP Fine-Tuning Pipeline

## Overview

This project provides a complete workflow for building and fine-tuning Arabic Natural Language Processing models using modern open-source LLM technologies.

The system covers the entire lifecycle from data preparation and annotation generation to supervised fine-tuning, evaluation, inference, deployment, and load testing.

The project leverages Qwen 2.5, Groq, OpenAI APIs, and LLaMA-Factory to create specialized models capable of performing structured information extraction and machine translation tasks.

---

## Objectives

The project aims to:

* Extract structured information from Arabic news articles.
* Translate Arabic content into multiple languages.
* Generate high-quality supervised fine-tuning datasets automatically.
* Fine-tune open-source language models using LoRA.
* Evaluate local and cloud-based language models.
* Deploy fine-tuned models efficiently using vLLM.
* Measure inference throughput and deployment performance.

---

## Features

### Information Extraction

Extracts:

* News title generation
* Keywords extraction
* News categorization
* Entity recognition
* Content summarization

Output is generated as structured JSON following predefined Pydantic schemas.

### Machine Translation

Supports translation from Arabic into multiple target languages while maintaining a structured JSON response format.

### Synthetic Dataset Generation

Uses powerful cloud models to automatically generate labeled datasets suitable for supervised fine-tuning.

### Supervised Fine-Tuning (SFT)

Creates instruction datasets compatible with LLaMA-Factory and trains domain-specific language models using LoRA.

### Model Evaluation

Compares outputs generated by:

* Base Qwen models
* OpenAI models
* Groq models
* Fine-tuned models

### Efficient Deployment

Supports:

* LoRA adapters
* vLLM serving
* OpenAI-compatible API endpoints

### Load Testing

Includes Locust-based benchmarking for production readiness evaluation.

## Dataset Generation Workflow

1. Load Arabic news articles.
2. Generate structured annotations using Groq.
3. Validate JSON outputs.
4. Store results as JSONL.
5. Convert data into instruction-tuning format.
6. Split into training and validation datasets.

---

## Fine-Tuning Pipeline

The project uses:

* Qwen/Qwen2.5-1.5B-Instruct
* LoRA Fine-Tuning
* LLaMA-Factory

Training configuration includes:

* LoRA Rank 64
* Cosine Learning Rate Scheduler
* Mixed Precision Training
* Evaluation During Training
* Hugging Face Hub Integration
